# CLAUDE.md — AI Assistant Guide for TFT Ninja

## Project Overview

TFT Ninja is a **Teamfight Tactics (TFT) educational documentation website** built with [Docusaurus v3.1.1](https://docusaurus.io/). It serves as "the missing TFT manual," providing guides on game mechanics, economy, items, and set-specific content. The site is deployed to GitHub Pages at the custom domain `tft.ninja`.

**Owner:** matt-landers
**Repository:** https://github.com/matt-landers/tft-ninja

## Tech Stack

- **Framework:** Docusaurus 3.1.1 (React-based static site generator)
- **Language:** TypeScript (~5.2.2)
- **UI Library:** React 18
- **CSS:** Infima (Docusaurus default) + CSS Modules
- **MDX:** @mdx-js/react 3.x (Markdown with JSX support)
- **Analytics:** Google Tag Manager (GTM-M4XM6LL9)
- **Node:** >=18.0 required

## Common Commands

```bash
npm start          # Start local dev server (hot reload)
npm run build      # Build static site to /build
npm run serve      # Serve built site locally
npm run typecheck  # Run TypeScript type checking (tsc)
npm run clear      # Clear Docusaurus cache (.docusaurus)
```

There are no test or lint commands configured in this project.

## Documentation Conventions

### Markdown Files (docs/)

All doc pages use YAML frontmatter with these fields:

```yaml
---
id: unique-identifier
title: Page Title
sidebar_position: 1
---
```

- `id` — unique page identifier used for linking
- `title` — displayed in sidebar and page heading
- `sidebar_position` — numeric ordering within parent category

### Category Configuration (docs/*/_category_.json)

Each directory in `docs/` uses a `_category_.json` file:

```json
{
  "label": "Section Name",
  "position": 2,
  "link": {
    "type": "generated-index",
    "description": "Section description."
  }
}
```

### Sidebar

The sidebar is auto-generated from the `docs/` directory structure (`sidebars.ts` uses `autogenerated` with `dirName: '.'`). Ordering is controlled by `sidebar_position` in frontmatter and `position` in `_category_.json`.

### Images in Docs

Item images use HTML `<img>` tags with AVIF format assets from `/img/tft/items/`:

```html
<img src="/img/tft/items/BFSword.avif" width="32" />
```

## Blog Conventions

## Component Architecture

### Homepage (`src/pages/index.tsx`)

The homepage renders:
1. **HomepageHeader** — Hero banner with site title, tagline, and CTA linking to `/docs/intro`
2. **HomepageFeatures** — Three feature cards (Game Mechanics, Common Tactics, Advanced Strategies)

### Styling

- **Global theme:** `src/css/custom.css` overrides Infima CSS variables
- **Color scheme:** Primary green (#2e8555 light / #25c2a0 dark)
- **Component styles:** CSS Modules (`*.module.css`) for scoped styles
- **Dark mode:** Supported via `[data-theme='dark']` CSS variable overrides
- **Utility:** `clsx` for conditional classNames

## Configuration

### Key Config Files

- **`docusaurus.config.ts`** — Site metadata, navbar, footer, plugins, theme
- **`sidebars.ts`** — Sidebar definition (auto-generated from filesystem)
- **`tsconfig.json`** — Extends `@docusaurus/tsconfig`, `baseUrl: "."`

### Important Config Details

- `onBrokenLinks: "throw"` — Build fails on broken internal links
- `onBrokenMarkdownLinks: "warn"` — Warns on broken markdown links
- `trailingSlash: false` — URLs do not have trailing slashes
- Edit URLs point to `https://github.com/matt-landers/tft-ninja/blob/main/`

## Deployment

- **Platform:** GitHub Pages
- **Custom domain:** `tft.ninja` (configured via `static/CNAME`)
- **Riot verification:** `static/riot.txt` must be preserved
- **Jekyll bypass:** `static/.nojekyll` ensures proper GitHub Pages serving

## Guidelines for AI Assistants

### When Adding Documentation

1. Create `.md` files in the appropriate `docs/` subdirectory
2. Always include frontmatter with `id`, `title`, and `sidebar_position`
3. Add `_category_.json` when creating new directories
4. Use AVIF format for new game images, placed in `static/img/tft/`
5. Keep content educational and accessible to TFT players of all skill levels

### When Adding Blog Posts

1. Create a directory: `blog/YYYY-MM-DD-slug/index.md`
2. Include frontmatter: `slug`, `title`, `authors`, `tags`
3. Add new authors to `blog/authors.yml` if needed
4. Co-locate images with the post in the same directory

### When Modifying Components

1. Use TypeScript for all React components
2. Use CSS Modules for component-scoped styles
3. Import Docusaurus utilities from `@docusaurus/*` and `@theme/*`
4. Use `@site/*` aliases for project-relative imports

### Build Validation

- Run `npm run build` to validate before committing — broken links will cause build failures
- Run `npm run typecheck` to check TypeScript types
- There are no automated tests; manual verification via `npm start` is the primary QA method

### Things to Preserve

- `static/CNAME` — breaking this breaks the custom domain
- `static/riot.txt` — Riot Games verification token
- `static/.nojekyll` — required for GitHub Pages
- Google Tag Manager integration in `docusaurus.config.ts`
